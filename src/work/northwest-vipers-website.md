<div class="case-sec"><div class="si">01 / the problem</div><div class="case-body" style="max-width:none">
<div style="max-width:64ch"><p>Three compounding issues made the old site useless as a club resource.</p></div>
<div class="trio"><div><div class="n">01</div><h4>Visual design broke down</h4><p>Yellow-on-black, all-caps throughout, no rhythm, text running to the viewport edge. The same problems at every screen size.</p></div><div><div class="n">02</div><h4>Content strategy collapsed</h4><p>Years of social-only focus left no archive, no depth, nowhere permanent to send a prospective member. Match reports vanished into a feed.</p></div><div><div class="n">03</div><h4>Only three of four squads</h4><p>One squad was absent entirely and the emphasis tilted toward another — reinforcing real internal tensions about which squads the club valued.</p></div></div>
</div></div>

<div class="case-sec"><div class="si">02 / process</div><div class="case-body">
<p>Three years as a club member was the primary research input — I understood the audience, the internal politics, the content needs and the maintenance constraints before a brief existed.</p>
<p class="muted">For a volunteer organisation, conventional research would have been friction without value. The right move was to build a demo and show it. Getting visible endorsement from club officers and squad captains before launch mattered for adoption — members look to them for the signal on whether to engage.</p>
</div></div>

<div class="case-sec"><div class="si">03 / technical decisions</div><div class="case-body" style="max-width:none">
<div style="max-width:64ch"><p>I replaced WordPress with <strong>11ty</strong>, a static site generator, hosted free on GitHub Pages. But the maintenance problem mattered as much as the design problem: a site only I could update would fail the same way the old one did.</p></div>
<div class="demos"><div><div class="a" style="color:var(--accent)">WHY 11TY OVER WORDPRESS</div><ul style="margin:6px 0 0;padding:0;list-style:none;display:flex;flex-direction:column;gap:8px"><li style="font-size:14px">Full control of layout, no theme constraints</li><li style="font-size:14px">Static output, zero hosting cost</li><li style="font-size:14px">No plugins, no update overhead</li><li style="font-size:14px">Maintainable long after my involvement</li></ul></div><div><div class="a" style="color:var(--accent)">CONTENT AS MARKDOWN</div><p>Content lives as plain Markdown files in the repo, with a clear folder structure. I showed non-technical members how to add a match report or photo gallery by editing a text file — no CMS, no editor UI, no WordPress knowledge.</p></div></div>
<div style="max-width:64ch"><p class="muted">This reframed the site from a design project into an operational tool the club could actually own.</p></div>
</div></div>

<div class="case-sec"><div class="si">04 / outcome</div><div class="case-body">
<p>The site launched and has published a match report and photo gallery for every match since. Its first real recruiting test — the Youth Kitted squad launch — worked: social drove awareness, the site provided the depth, and the squad is now training.</p>
<p class="muted">The site-to-social coordination still needs work — post timing isn't aligned yet. That's an identified next step, not a design failure.</p>

01 / what I do

Three things I'm hired for

01

Machine-readable architecture

Token naming, prop conventions, and documentation structured so AI prototyping tools can generate against the system accurately — not just a Figma library, but infrastructure that serves human designers and automated pipelines alike.

02

Governance at scale

Contribution frameworks, adoption metrics, and deprecation processes that keep a system coherent across 60 brands or 150+ products. The hard part isn't building components — it's keeping them reliable as the organisation ships against them.

03

Design-to-code pipeline

StyleDictionary, Figma Variables, CSS custom properties, W3C DTCG tokens — bidirectional sync between design and engineering. A custom Figma plugin built with Claude that automated token transforms across Figma, code, and docs at once.

04 / about

Twelve years making design systems load-bearing

I've designed for the web since 2000 — the last twelve years focused on design systems. I've built the design function from scratch at a startup, led the enterprise system at AVEVA across a 20-person UX org, and managed a five-designer team scaling 60 brands at Atolls. I design and write front-end — HTML/CSS, React, Vue, Angular — so the spec and the code never drift apart.

Accessibility (WCAG 2.1) runs through every role. Off the clock I chair and coach an American football club in the northwest of Ireland — governance, budgets, and a website I designed and built myself.
